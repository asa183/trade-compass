# Trade Compass - 開発状況と今後のプラン

## 📝 これまでに実装したこと (完了済)

### 1. 確信度の視覚的な「濃淡」表現 (Color Coding)
- 確信度の数値を直感的に判断できるよう、スコアに応じた動的なカラー設定（`success`, `accent`, `warning`, `danger`）を実装しました。
- ホーム画面、ディール詳細、バスケット一覧・詳細にて、数値の高さに応じたバリアント表示を適用済です（`lib/ui.ts` ）。

### 2. オンボーディングの操作性改善（スキップ導線）
- オンボーディング（診断）画面の右上に「**トップへ戻る** ✕」ボタンを配置しました。
- これにより、途中でホーム画面を確認したくなった場合や、後回しにしたい場合に、仮のデフォルト値でセットアップをスキップしてダッシュボードに遷移できるようになりました。

### 3. 操作画面・UX全体の向上
- **トースト通知の共通化**: アプリ全体で即座に状況を伝えるグローバルなトーストUI基盤を構築しました。
- **手動更新**: ホーム画面等で、ブラウザリロードに依存せず市場データ・ディールを最新化するリフレッシュ機能を搭載しました。
- **ペーパートレードの決済UI**: ワンタップで決済を完結できるよう、現在のライブ価格の自動入力や理由選択を整備しました。
- **バスケット探索**: 50種類のバスケットを検索・テーマフィルタリングできるUIを追加しました。

### 4. 環境依存パパッケージの修正 (NEW)
- デプロイの際にエラーになっていた `dotenv` の欠落を修正し、正常にデプロイパイプラインが実行できるようにしました。

---

## 🚀 ここからのプラン (Next Steps)

### 1. ダッシュボードの動的集計ロジックの完全実装 (推奨度：高)
現在のペーパートレード履歴やディールの成功・失敗状況をもとに、**実績（ダッシュボード）**ページの各種メトリクス（勝率、想定損益、ユーザーの傾向スコアなど）を動的に表示する機能の精度を高めます。

### 2. マイナートラブル・実機フィードバックの修正
本番環境での動作確認や、ペーパートレード時のデータ整合性・価格ズレの検知など、実際にアプリを操作した上で発生する細かなフィードバックの回収とUI調整を行います。

### 3. ディール・市場レジームの定期自動更新機能の強化
現在搭載されているAPIルート（`cron/route.ts` や `market/quotes`）を拡張し、ユーザーが手動で更新しなくてもバックグラウンドで高精度かつ高速に最新の推奨ディールを計算する仕組みを安定稼働させます。

### 4. コンテンツ・推奨銘柄の拡充対応
読み込み速度を維持しつつ、ユーザーにあわせて最大5つ表示する「おすすめ銘柄」アルゴリズムの調整および、データ根拠（AI生成テキスト）の表示の洗練を図ります。
